# Convert Regression Fixture

`test-convert-regression` pins the `convert svfdrive` output for a
deterministic S32_LE mono input generated by `svfdrive_input.c`. The expected
SHA256 is intentionally tied to the GitHub Actions Ubuntu x86_64 stock GCC
build used by CI, because `convert.o` and the effect registry build with
`-ffast-math`.

If the CI compiler, runner architecture, or DSP flags change, regenerate this
hash only after confirming and documenting the output difference.
