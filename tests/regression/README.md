# Convert Regression Fixture

`test-convert-regression` pins the `convert svfdrive` output for a
deterministic S32_LE mono input generated by `svfdrive_input.c`. CI enforces
the GitHub Actions Ubuntu 24.04 x86_64 stock GCC hash
`73b7f69ceb81451daa69e0809f03257208033bf10709c6135ed49224e9ccb321`,
because `convert.o` and the effect registry build with `-ffast-math`.

Local non-CI runs still print the computed hash, but they do not fail solely
because the compiler differs from the CI runner. For example, Fedora GCC 15
produced `d47942152608df076ac3af64cc9df6b809747011bf0cea72519c50b906db00de`
when this fixture was added.

If the CI compiler, runner architecture, or DSP flags change, regenerate this
hash only after confirming and documenting the output difference.
